The InputStream,OutputStream,Reader,Writer is not closed after use.

On UNIX-like platforms, the OS places a limit on the number of open file descriptors that a process may have at any given time. When the this number is exceed, operations about File/Socket/Thread will fail with an IOException:'Too many opened file'.

An exception those pure in-memory IO resources, such as ByteArrayInputStream, and those resource which is managed by container, such as OutputStream from HttpServletRequest which is managed by the servlet container.
