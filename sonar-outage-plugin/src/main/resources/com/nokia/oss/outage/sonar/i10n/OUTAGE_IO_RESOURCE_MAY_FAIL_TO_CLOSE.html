The resource may fail to close due to the exception of previous IO operation.

Take the following code as an example


    public static void copy( File src, File dest ) throws IOException
      {
       FileInputStream inputStream = null;
       FileOutputStream outputStream = null;
       try
       {
         ...//bossiness logic
       }
        finally
       {
         if( inputStream != null )
           {
             inputStream.close();
            }
         if( outputStream != null )
           {
            outputStream.flush();
            outputStream.close();
           }
       }
    }

If an IOException is thrown when closing the inputStream, then the outputStream is not closed at all
